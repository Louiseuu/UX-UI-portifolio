Physical Inventory App

A 0→1 product design for SAP's new physical inventory module — built for two roles: Operatives executing on-floor counting tasks, and Clerks managing counting plans and confirming final results. Designed from scratch to simplify the end-to-end workflow across both roles. UI has been redesigned from the original for portfolio presentation purposes.

01 Research

Understanding the real users
behind warehouse counting

Before touching any design tool, we needed to deeply understand how physical inventory was actually done and who was doing it. We ran semi-structured interviews with warehouse operatives and logistics clerks, then distilled the findings into personas and a To-Be journey map.

Interview Setup
Method
Semi-structured User Interviews
Each session covered 4 topic areas: Timing & Planning, Counting Process, Review & Validation, and a "Magic Button" open question — designed to surface functional needs and latent frustrations.
Interview Questions — 4 Topic Areas
01
Timing & Planning
How do you plan and assign counting tasks?
  • How do you assign tasks to the right operatives?
  • Which exception cases trigger an ad-hoc count?
  • How strict are planned count deadlines?
02
Counting Process
What happens on the warehouse floor?
  • What could block you mid-count and interrupt the process?
  • How do you transmit results to the system?
  • What happens when there are zero items at a location?
03
Review & Validation
How do you handle discrepancies?
  • What happens when the count doesn't match the system?
  • How is a recount decision triggered?
  • What are consequences of inventory corrections?
04
"Magic Button"
What would you fix first?
"If you had a magic button that could solve any issue in the current physical inventory process, what would it do?"

02 Ideation

Turning insights into
design opportunities

With research complete and our personas defined, we synthesised the journey map findings into "How Might We" questions — reframing each pain point as a concrete design challenge before moving into solutions.

How Might We Questions
HMW #1
How might we help warehouse operatives complete counting tasks without interruption or confusion about location and scope?
HMW #2
How might we give managers real-time visibility into inventory progress and discrepancies without extra reporting steps?
HMW #3
How might we reduce data entry errors when operatives submit count results from the warehouse floor?
HMW #4
How might we make discrepancy escalation faster, structured, and automatically visible to the right people?
Key User Stories
Operative — Execution
As an Operative, I want to view my assigned tasks (counts, recounts, investigations) so I know exactly what to work on next.

As an Operative, I want to update task status in real time so the dashboard reflects actual progress.
Operative — Discrepancy
As an Operative, I want to report discrepancies directly in the app and attach photo evidence so investigations start without delay.

As an Operative, I want clear SOP references per task so I always follow the correct process.
Manager — Monitoring
As a Manager, I want to see KPIs: accuracy %, discrepancies, backlog, SLA compliance so I can evaluate performance and identify risks.

As a Manager, I want early alerts on critical issues so I can intervene before they escalate.
Shared Needs
As a user, I want a simple, intuitive interface so I can quickly understand what needs attention.

As a user, I want role-based access so I only see information relevant to my responsibilities.

04 Validation

Testing with real users

We conducted moderated usability sessions with customers and partners using a lo-fi prototype, applying think-aloud protocol to surface pain points early before investing in visual design.

Findings & Design Responses
# Finding User Feedback Design Response
1 Create flow was unclear — storage bin name alone was not enough to identify the right bin "Can I see the bin's status, stock movement, or other context to help me decide?" Redesigned the create entry point around a Storage Bin List — each bin surfaces rich contextual info (status, movement, availability) to guide task creation
2 Counting flow was too fragmented — excessive navigation broke the working rhythm "I have to keep tapping back and forth between tasks. There's no sense of flow — I just want to work through it all at once." Bundled tasks and exposed item-level detail directly in the list, grouped by storage bin — mirrors how operatives actually move through a warehouse zone
3 Confirming results was tedious — tasks had to be reviewed and posted one by one "I have to open each task separately to check and confirm. It's the same thing over and over." Applied the same item-level structure as counting — all items consolidated in a single view for side-by-side comparison, with batch confirmation for tasks with no discrepancy
05 Hi-Fi Prototype

Bringing the design to life

Design decisions were validated with the PM and engineering team for feasibility before moving to high-fidelity. The screens below reflect both user needs and technical constraints, and have been redesigned for portfolio presentation purposes.

Part 1 — Create Physical Inventory Task
Storage Bin List
Main screen — Clerk / Manager view
Bin Detail
View counting history & stock info — click → to navigate. Supports discrepancy tracing when exceptions arise.
Unavailable Popover
Click "Unavailable for counting" tag — shows reason & related in-progress documents
Create Dialog
Click "Create Physical Inventory task" — confirm type, date & description
Part 2 — Counting Physical Inventory
Count Task List
Main screen — overview of all counting tasks with progress. Filter by number of items helps temporary workers quickly pick up small-volume counting tasks.
Task Detail
Click → to enter — view items to be counted, click "Count" to start
Counting Mode
Click "Count" to activate — enter quantities inline, save as draft anytime. A clear indicator appears on each item once a quantity is entered, making counting status immediately visible at a glance.
Partial Save
Counted items move to bottom — remaining items stay on top for easy continuation
Part 3 — Confirm Result & Report Discrepancy
Confirm Count Results
Review all counted items with difference quantities highlighted. Batch-post zero-difference items by filtering Difference Quantity = 0 — eliminating unnecessary one-by-one confirmation. New Stock and Zero Count tags surface common misplacement patterns, helping managers quickly match discrepancies across bins.
Post Success
Dialog provides a direct link to Reconcile Stock app for discrepancy reporting.
Reconcile Stock
Click "Reconcile Stock" in the dialog to jump to the Reconcile Stock app — report the discrepancy to the ERP system (SAP S/4HANA) and post the difference quantity.
Part 4 — Overall Dashboard
Inventory Overview
KPI cards prioritise the metrics managers check first — accuracy, today's progress, and active discrepancies — reducing the need to dig into detail views for routine status checks. Daily and weekly progress bars provide immediate orientation on where the team stands against targets.
Discrepancy Analysis & Early Warnings
Discrepancies are categorised by root cause and mapped to storage zones, making it easier to identify patterns rather than reviewing individual items. High-risk areas are visually flagged for immediate attention. The Early Warnings section surfaces proactive alerts before issues escalate, each paired with predicted impact and a recommended action.
Inventory Condition & Movement
ABC classification gives managers a prioritisation framework — high-value items stay in focus without manual filtering. Movement tracking is scoped per storage bin with switchable time ranges, enabling targeted investigation of activity spikes without leaving the dashboard.

Validating with real users

We conducted moderated usability testing on the high-fidelity prototype with participants performing 3 core task scenarios. Results confirmed mid-fi decisions and surfaced final refinements.

Test Setup
Method
Moderated Usability Testing
Remote moderated sessions using think-aloud protocol. Each participant navigated the hi-fi prototype independently while verbalising their thought process.
Scenarios Tested
3 Core Tasks
1. Create a new counting task
2. Complete a count task end-to-end
3. Report a discrepancy with photo evidence
Participants
Customers & Partners
SAP customers and partners with hands-on experience in inventory and warehouse operations.
Final Findings
Task Result Observation Follow-up Opportunity
Create a new counting task Partial Storage bin contextual info helped participants make more confident decisions. However, several noted that manually creating tasks for recurring cycles felt repetitive and time-consuming. Automate task creation via counting rules — reduce manual setup for recurring cycles
Complete a count task end-to-end Pass Task bundle structure and item-level grouping by storage bin were intuitive. Participants moved through the flow without backtracking. Some raised questions about how tasks reach operatives in the field, particularly in large warehouses. Mobile work queue to allow managers to assign and dispatch tasks directly from the warehouse floor
Report a discrepancy Partial Participants identified and filtered discrepant items without difficulty. The Reconcile Stock deep-link was well received. However, when root causes were ambiguous, participants paused and expressed uncertainty about what to do next. AI assistant to surface likely root causes and suggest corrective actions based on discrepancy patterns

Reflection
Balancing standardisation with usability was the biggest challenge throughout this project. Within the constraints of technical architecture and system conventions, the core design work was about reconstructing the real-world mental model of warehouse operatives — not just mapping UI onto existing flows. For rules and behaviours dictated by the system, the priority became communicating them clearly to users, and finding every opportunity to surface shortcuts and cross-feature navigation so the experience feels connected rather than siloed.
— Designer Reflection
